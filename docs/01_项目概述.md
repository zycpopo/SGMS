# 学生课程成绩管理系统 — 项目总说明

## 项目名称

**学生课程成绩管理系统：基于 MASM 的 DOS 16 位汇编 CRUD 应用程序**

## 项目定位

本项目是一个运行在低版本 DOS 环境下的学生课程成绩管理系统。程序使用 **MASM 语法** 编写，面向 **8086 / 16 位实模式 / DOS int 21h** 环境，能够在 DOSBox、FreeDOS 或真实低版本 DOS 环境中运行。

这不是一个简单的菜单程序，而是一个完整的数据管理系统，包含：

- CSV 文件读写（score.txt）
- 完整的 CRUD 操作（增删改查）
- 逻辑删除（标记删除，数据保留）
- 成绩统计（平均分、最高分、最低分、分布）
- 文本字符柱状图可视化
- 定长字段内存存储
- 格式化表格显示

## 运行环境要求

推荐环境：Windows / Linux 主机 + DOSBox 或 DOSBox-X + MASM.EXE + LINK.EXE。

程序需要兼容低版本 DOS，因此必须遵守：

- 使用 `.MODEL SMALL`
- 使用 `.STACK 100H`
- 使用 `.DATA` 和 `.CODE`
- 只使用 AX、BX、CX、DX、SI、DI、BP、SP
- 不使用 EAX、EBX、ECX、EDX
- 不使用 `.386`、`.MODEL FLAT`、Windows API
- 程序界面字符串使用英文 ASCII（保证在标准 DOSBox 中正常显示）
- 源码注释可使用中文

## 项目核心目标

本项目的核心目标不是追求算法复杂，而是追求汇编基础扎实、DOS 中断使用规范、程序结构清晰、功能完整、演示稳定。重点展示 DOS 文件 I/O、内存数据管理、模块化过程设计以及文本可视化能力。

## 程序功能总览

### 文件存储与加载

启动时自动从 `score.txt` 读取已有数据。如果文件不存在则自动创建。退出时将所有有效记录写回文件（已删除的不写入），字段尾部空格自动去除。

### CRUD 操作

- **添加**：输入学号、姓名、课程、成绩（0-100 校验），追加到内存数组
- **查找**：按学号或姓名精确匹配，格式化表格显示结果
- **修改**：按学号查找，显示当前记录，输入新成绩更新
- **删除**：逻辑删除（标记位设为 1），数据物理保留但不再显示/统计/保存

### 统计功能

- 有效记录总数
- 平均分（总分 ÷ 有效记录数）
- 最高分、最低分
- 分数段分布：90-100、80-89、70-79、60-69、<60

### 柱状图

- 用 `#` 字符绘制水平柱状图
- 自动缩放：当某段人数超过 50 时，每个 `#` 代表多个人
- 每段显示对应人数

## 数据格式

`score.txt` 使用 CSV 格式：

```
学号,姓名,课程,成绩
20240001,Alice,Calculus,85
20240002,Bob,English,92
```

## 内存记录布局

每条记录 43 字节：

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 8B | 学号 | 右补空格 |
| 8 | 16B | 姓名 | 右补空格 |
| 24 | 16B | 课程 | 右补空格 |
| 40 | 2B | 成绩 | Word 类型 (0-100) |
| 42 | 1B | 删除标记 | 0=有效, 1=已删除 |

最大容量：100 条记录（共 4300 字节）。

## 编码风格要求

- MASM 兼容语法，文件名为 `SCORE.ASM`
- 主过程命名为 `MAIN PROC FAR`
- 用子程序（PROC）拆分功能，不把所有逻辑堆在 MAIN 中
- 字符串以 `$` 结尾，方便 DOS `int 21h AH=09h` 输出
- 输入使用 DOS `int 21h AH=01h`（单字符）或 `AH=0Ah`（缓冲输入）
- 文件操作使用 DOS `int 21h AH=3Ch/3Dh/3Eh/3Fh/40h`
- 定义 PRINT_STR 宏简化字符串输出

## 不允许出现的问题

不要生成 32 位汇编、Linux 汇编、NASM 语法、Windows API 程序、中文 DOS 界面字符串、一屏挤满文字的界面、没有注释的大段代码、只能在高版本 DOS 跑的代码。

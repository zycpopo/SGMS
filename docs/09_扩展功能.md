# 可视化与交互功能说明

## 概述

学生课程成绩管理系统，虽然主要定位为数据管理应用，但在 DOS 文本模式约束下，实现了多种可视化和交互功能。本文档详述这些功能及其技术实现。

## 功能清单

### 1. 文本柱状图（数据可视化）

**技术实现：**
- 使用 `#` 字符渲染水平柱状图
- 5 段柱分别代表：90-100、80-89、70-79、60-69、<60
- 每个 `#` 代表 1 人或多个人（自动缩放）
- 通过 DOS INT 21H AH=02H 逐字符绘制

**缩放算法：**
```text
IF 最大人数 > 50:
    scale = 最大人数 / 50 + 1
ELSE:
    scale = 1
柱长度 = 人数 / scale  (人数 > 0 时最小为 1)
```

**显示效果：**
```text
+========== Score Histogram ==========+
  (each # = 1 person(s))

  90-100: ##### ppl
  80-89 : #### ppl
  70-79 : ### ppl
  60-69 : ## ppl
  <60   : # ppl
```

**汇编实现要点：**
```asm
DRAW_ONE_BAR PROC
    ; AX = 人数, BX = 缩放比例
    CMP  AX, 0
    JE   DOB_ZERO           ; 人数为 0 则跳过
    XOR  DX, DX
    DIV  BX                 ; AX = 柱长度
    CMP  AX, 0
    JNE  DOB_LOOP
    MOV  AX, 1              ; 非零段最少 1 个 #
DOB_LOOP:
    MOV  CX, AX
DOB_CHAR:
    MOV  AH, 02H
    MOV  DL, BAR_CHAR       ; '#'
    INT  21H
    LOOP DOB_CHAR
    ...
DRAW_ONE_BAR ENDP
```

### 2. ASCII 边框界面

**技术实现：**
- 统一 40 字符宽度边框
- 使用 `+`、`-`、`|`、`=` 字符绘制
- 标题在边框内居中
- 统计和柱状图区域有分段标题

**边框样式：**

| 边框类型 | 用途 |
|----------|------|
| `+====...+` | 分段标题（统计、柱状图） |
| `+----...+` | 菜单边框 |

### 3. 格式化表格显示

**技术实现：**
- 定宽列输出
- 表头行加分隔线
- 数据列对齐
- 字段空格填充显示

**列布局：**
```text
ID        Name             Course           Score
--------  ----------------  ----------------  -----
XXXXXXXX  XXXXXXXXXXXXXXXX  XXXXXXXXXXXXXXXX  XXX
 8 字符     16 字符           16 字符          3 位数字
```

**汇编实现：**
```asm
PRINT_FLD PROC
    ; 用 AH=02H 从 [SI] 打印 CX 个字符
PFLD_LP:
    MOV  DL, [SI]
    MOV  AH, 02H
    INT  21H
    INC  SI
    LOOP PFLD_LP
    RET
PRINT_FLD ENDP
```

### 4. 状态反馈系统

**技术实现：**
- 文本颜色编码前缀：`[OK]`、`[INFO]`、`[ERROR]`
- 每次 CRUD 操作后即时反馈
- 输入校验失败时重新提示
- 查找结果显示匹配计数或"未找到"

**消息分类：**

| 前缀 | 建议颜色 | 用途 |
|------|----------|------|
| `[OK]` | 绿色 | 操作成功 |
| `[INFO]` | 蓝/青色 | 状态信息 |
| `[ERROR]` | 红色 | 错误状态 |

### 5. 交互式菜单导航

**技术实现：**
- 单键菜单选择（无需按回车）
- 无效按键静默忽略（菜单重新显示）
- 清晰的选项编号（0-7）
- 即时功能分发

**交互流程：**
```text
主菜单 → [按键] → 功能执行 → 返回菜单
   ↑                            |
   └────────────────────────────┘
```

### 6. 数据持久化与文件可视化

**技术实现：**
- 数据通过 `score.txt` 在程序重启间持久化
- CSV 格式人类可读，可外部编辑
- 首次运行自动创建文件
- 退出时保存确认

## DOS 功能调用总览

### DOS 中断（INT 21H）

| 功能号 | AH | 用途 |
|--------|----|------|
| 带回显字符输入 | 01H | 菜单选择 |
| 字符输出 | 02H | 柱状图 `#` 字符、字段打印 |
| 字符串输出 | 09H | 所有文本显示（通过 PRINT_STR 宏） |
| 缓冲输入 | 0AH | 多字符输入（学号、姓名、课程、成绩） |
| 创建文件 | 3CH | 保存/覆盖 score.txt |
| 打开文件 | 3DH | 加载 score.txt |
| 读文件 | 3FH | 读取文件内容到缓冲区 |
| 写文件 | 40H | 写入 CSV 行 |
| 关闭文件 | 3EH | 关闭文件句柄 |
| 程序退出 | 4CH | 返回 DOS |

## 可选增强（本项目未实现，供后续扩展）

对于需要显式"多媒体"功能的场景，可考虑以下增强：

### 彩色界面增强
将 `PRINT_STR` 替换为 BIOS INT 10H 彩色输出：
- 标题 → 亮黄色（0Eh）
- 边框 → 亮青色（0Bh）
- 成功消息 → 亮绿色（0Ah）
- 错误消息 → 亮红色（0Ch）

### 音效
使用 PC 扬声器（端口 61h/43h/42h）：
- 添加/修改/删除成功 → 短促提示音
- 输入错误 → 低音警告
- 程序退出 → 双音和弦

### 屏幕动画
- 功能页面间清屏过渡
- 字符逐字打印效果（延迟输出）

## 总结

本项目通过 DOS 汇编语言实现了一个具有完整交互体验的数据管理系统，包含：
- ASCII 艺术边框和表格界面
- `#` 字符柱状图可视化
- 即时状态反馈系统
- 交互式菜单导航
- CSV 文件数据持久化

项目侧重于展示 16 位汇编在数据管理和文本可视化方面的能力，所有功能在不依赖外部库的情况下通过 DOS 中断实现。

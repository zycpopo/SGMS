# 测试与验收标准

## 编译验收

必须能在 DOSBox 中执行：

```dos
MASM SCORE.ASM;
LINK SCORE.OBJ;
SCORE.EXE
```

不能出现 Undefined symbol、Syntax error、Phase error、Segment error、32 位寄存器错误、无法链接。

## 启动验收

**首次运行（无数据文件）：**
1. 删除 `score.txt`（如存在）
2. 运行 `SCORE.EXE`
3. 预期：显示 `[INFO] score.txt not found, created new file.`
4. 验证 `score.txt` 已创建
5. 主菜单出现

**有数据文件：**
1. 确保 `score.txt` 存在且包含有效 CSV 数据
2. 运行 `SCORE.EXE`
3. 预期：显示 `[OK] score.txt loaded.`
4. 主菜单出现

## 菜单验收

- 菜单显示全部 8 个选项（0-7）并带边框
- 输入 0-7：导航至对应功能
- 输入 8、9、字母等无效输入：菜单静默重新显示（不崩溃、不报错）

## 添加记录验收

**测试：添加一条有效记录**
1. 选择 `1. Add Record`
2. 输入：ID=`20240001`、Name=`Alice`、Course=`Calculus`、Score=`85`
3. 预期：`[OK] Record added.`
4. 选择 `7. List All Records` → 验证 Alice 出现

**测试：成绩校验**
1. 选择 `1. Add Record`
2. 输入成绩 `150` → 预期重新提示
3. 输入成绩 `-5` → 预期重新提示
4. 输入成绩 `85` → 接受

**测试：容量限制**
1. 添加 100 条记录（或临时修改 SCOUNT 做快速测试）
2. 尝试再添加一条 → 预期：`[ERROR] Capacity full (100 max)!`

## 查找验收

**测试：按学号查找**
1. 选择 `2. Search Record`
2. 选择 `1`（按学号）
3. 输入已有学号（如 `20240001`）
4. 预期：以表格格式显示匹配记录

**测试：按姓名查找**
1. 选择 `2. Search Record`
2. 选择 `2`（按姓名）
3. 输入已有姓名（如 `Alice`）
4. 预期：显示匹配记录

**测试：查找不存在**
1. 搜索不存在的学号（如 `99999999`）
2. 预期：`[INFO] No matching record found.`

**测试：多条匹配**
1. 添加同一学号但不同课程的多条记录
2. 按学号搜索 → 预期显示所有匹配记录

## 修改验收

1. 选择 `3. Modify Score`
2. 输入已有学号
3. 当前记录显示
4. 输入新成绩（如 `95`）
5. 预期：`[OK] Score updated.`
6. 列出全部记录 → 验证成绩已更新

**测试：修改不存在的记录**
1. 输入不存在的学号
2. 预期：`[INFO] No matching record found.`

## 删除验收

1. 选择 `4. Delete Record`
2. 输入已有学号
3. 当前记录显示
4. 预期：`[OK] Record deleted.`
5. 列出全部记录 → 已删除记录不再显示
6. 保存并退出 → 检查 `score.txt` → 已删除记录不在文件中

**测试：删除不存在的记录**
1. 输入不存在的学号
2. 预期：`[INFO] No matching record found.`

## 统计验收

1. 手动添加若干条已知成绩的记录（如 85、92、78、60、55）
2. 选择 `5. Statistics`
3. 验证：
   - 有效记录数正确
   - 平均分正确（整数除法）
   - 最高分和最低分正确
   - 各分数段分布计数正确

**测试：空数据集**
1. 以空数据集启动
2. 选择 `5. Statistics`
3. 预期：`[INFO] No records.`（不崩溃）

## 柱状图验收

1. 添加成绩分布多样的记录
2. 选择 `6. Histogram`
3. 验证：
   - 标题显示
   - 比例说明显示（如 `each # = 1 person(s)`）
   - 各段 `#` 字符数量正确
   - 人数显示在每段后面

**测试：自动缩放**
1. 在某一段添加 60+ 条记录
2. 选择柱状图
3. 验证比例自动增大（如 `each # = 2 person(s)`）
4. 验证柱状图适合屏幕宽度

**测试：空分布**
1. 确保某段人数为 0
2. 验证该段显示 `0 ppl`，没有 `#` 字符或显示为空

## 显示全部验收

1. 添加多条记录
2. 选择 `7. List All Records`
3. 验证：
   - 表头含列名
   - 所有有效记录以对齐列格式显示
   - 已删除记录**不显示**

## 保存并退出验收

1. 添加/修改/删除一些记录
2. 选择 `0. Save & Exit`
3. 预期：显示 `[OK] Saved to score.txt.` 然后显示 `Goodbye!`
4. 检查 `score.txt` 内容 → 验证 CSV 格式正确
5. 重新运行程序 → 验证保存的数据正常加载

**测试：有删除记录的保存**
1. 删除部分记录但不退出
2. 保存并退出
3. 检查 `score.txt` → 已删除记录不存在于文件中

## 文件格式验收

保存后，`score.txt` 应包含：

```
20240001,Alice,Calculus,85
20240002,Bob,English,92
```

**不应**出现：

```
20240001   ,Alice           ,Calculus        ,85
```
（CSV 字段中不应有尾部空格）

## 兼容性验收

源码中不能出现：
- `EAX`、`EBX`、`ECX`、`EDX`、`ESI`、`EDI`、`EBP`、`ESP`
- `.386`、`.486`、`.MODEL FLAT`
- `org 100h`（COM 风格）
- `int 80h`（Linux）
- Windows API 调用
- 中文界面字符串
- Unicode 边框字符

## 推荐测试流程

1. 删除 `score.txt`，冷启动
2. 添加 5 条不同成绩的记录
3. 列出全部 → 验证 5 条全显示
4. 按学号查找 → 验证匹配
5. 按姓名查找 → 验证匹配
6. 修改一条记录的成绩 → 验证修改成功
7. 删除一条记录 → 验证不再显示
8. 统计 → 验证计算正确
9. 柱状图 → 验证视觉输出
10. 保存并退出 → 验证文件内容
11. 重新启动 → 验证数据持久化
12. 再添加一条 → 验证计数正确递增
13. 退出 → 完成
